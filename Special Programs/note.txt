//String is basically an object that represents immutable sequence of characters.
//Creation of String objects can be done in 2 ways.
//1. By String Literal.
//2. Using New Keyword.
 For example of string literal.
 String str = "Hello";  only one object is created.
 Each time a string literal is created, the JVM checks the String constant pool first.
 If the string is already created, then it returns the reference of the pooled 
 instance and if is not created then an instance is created and placed in SCP.

 For example of new keyword.
 String st = new String("Hello"); two objects are created and one reference variable.
 In this JVM creates a object in non pool heap memory and it's literal for example "hello"
 is placed in SCP. The variable 'st' will refer to the object in heap (non pool).

 String methods.
 Java class provides us many methods to perform operations on strings 
 such as length,compareTo,equals,concat,substring etc

 example of length:
 String str = "Hello";
System.out.println("The length of "+str+" is "+str.length());//gives answer 5

example of compareTo:
String st = "hello";
String str = "hello";
System.out.println(st.compareTo(str)); returns 1 if str = "hell", returns 0 if equal, 
returns -1 if str = "helloi"

example of substring:
String str = "Hello";
System.out.println("Substring = "+str.substring(2)); gives answer llo. 
Note: we can also give 2 parameters in substring method. Here one parameter means only the 
start index whereas, if we give 2 parameters it means that it has both start and end index.

StringBuffer:
- StringBuffer Represents a mutable sequence of characters. 
- It allows us to modify the content of a string without creating a new object, thereby saving memory. 
- Useful for concatenating and modifying strings, as it provides us methods for appending,deleting,inserting characters. 
- StringBuffer is thread-safe, making it suitable for use in multithreaded environments. 
- Additionally, StringBuffer dynamically grows in size as per the requirement. 

Methods of StringBuffer:

Note:- We can use this methods directly in system.out.println(); but i wrote like this for 
understanding.

1. append() method:
StringBuffer s = new StringBuffer("Java");
s.append("script");
System.out.println(s);//Javascript

2. replace() method:
StringBuffer str = new StringBuffer("Jooa");
str.replace(1,3,"av");// 1 represents start index, 3 represents end index
System.out.println(str);// Java

3. insert() method:
StringBuffer s = new StringBuffer("hello");
s.insert(5,"java");//5 represents start index
System.out.println(s);// hellojava

4. reverse() method:
StringBuffer s = new StringBuffer("Java");
s.reverse();
System.out.println(s);//avaJ

5. delete() method:
StringBuffer s = new StringBuffer("Javascript");
s.delete(0,4);
System.out.println(s);//script

Call By Value:
- In java, call by reference is not possible but call by value is possible.
- In call by value, the method receives a copy of the actual argument's value. 
- Changes made to the parameter within the method do not affect the original variable.
eg:-
class demo{
int a = 10;
void show(int a){
a = a + 20;
}
public static void main(String ar[]){
demo t = new demo();
System.out.println("Before calling the function: "+t.a);//10
t.show(30);
System.out.println("After calling the function: "+t.a);//10
}
}

Call By Reference:
eg:-
class demo{
int a = 10;
void show(demo t){
t.a = t.a + 20;
}
public static void main(String ar[]){
demo t = new demo();
System.out.println("Before calling the function: "+t.a);
t.show(t);
System.out.println("After calling the function: "+t.a);
}
}

Method overriding:
- When a subclass provides a specific implementation of a method which is already defined in its superclass, allowing the subclass to modify or extend the behavior.
- Method overriding is used to achieve runtime polymorphism.
- There are 3 rules of method overriding.
  -->1.There must be a IS-A relationship (inheritance).
  -->2.The method must have same name as in parent class.
  -->3.The method must have same parameter as in parent class.

this keyword:
1.used to refer to current class instance variable:

demo t = new demo();
